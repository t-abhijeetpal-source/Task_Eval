## Where the 4 missing points are (honest)
- **Documentation accuracy −1:** `amount` is still a float; correct for the demo, but money should be
  integer minor units. Documented (AUDIT-A3-011) and pinned by the Rust boundary test, not yet migrated.
- **Resilience −1:** single-node file queue + SQLite. WAL + engine timeouts mitigate; true HA needs the
  broker + Postgres migration (AUDIT-A3-027/028, `BROKER_MIGRATION.md`).
- **Observability −2:** structured JSON logs + `request_id` are in place, but there is no `/metrics`
  endpoint or load-test harness yet (AUDIT-A3-016/018).

These are deliberately left as documented, tracked backlog rather than claimed-done — consistent with
the constitution's "every metric claim requires captured output."
